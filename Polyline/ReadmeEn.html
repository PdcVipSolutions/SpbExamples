Motivation

This is an example of the evolution of the project development. A very easy game has been chosen as the basic idea. We do not know the initial name of this game so we call it Polyline.

Each development step represented as the release, so you can see step by step the evolution of the application. Each release expands the functionality of the application, moving along the way the code is extended, modified and restructured or only restructured and modified.

Looking carefully on the code of each release you can see technique of the using of the Visual Prolog and advanced tools. It can be used to study the Visual Prolog features.

Game description

The game is played on an M x N board.

Game rules

  • Each player on his turn adds a new point. The point must prolong the existing Polyline (or start the Polyline, if it doesn't exist) in the vertical or hirisontal direction only.
  • The resulting Polyline must be non-interrupted and each move can be taken at any of end of the line.
  • The player, which locks the line (the point is set on the polyline), wins.

Building notes

  1. Releases 1-9 projects may be built one by one without any tricks.
  2. Since Release 10 all projects use SpbSolutions IDE variable, which must refere to the SpbVipTools directory.
  3. Since Release 11 some projects demonstrate building libraries, thuse user must have Microsoft lib.exe program tool (see details in the release appropriate Readme.htm file).
  4. Projects placed to the Release 14 (Polyline14) PzlBasedApplicaions folder need to run the registration procedure after buiding all projects placed to the Containers directory. This must be done using the Programs\pzlBasedApplication\RegisterComponents.cmd command file.
  5. Some Projects since Release 12 demonstrate and use the DLL creation. So these projects must be built using VIP CE only (please see details the file VersionReference.html).
  6. All Polyline-related projects create the executable programs at Polyline\Execute folder.

Release 12 (Folder PolyLine12)

The Release 11 projects family had the problem - the overloading of the call stack on some values of the multiplication TheSizeOfTheBoard*AmountOfRounds and appropriate breakdown of the application.

The purpose of the Release 12 is to solve this problem.

The set of projects

The set of project in Release 12 differs from the set of projects of the Release11:

  • First, Release 12 doesn't contain Console-based projects.
  • Second, project ObjectWin contains now three variants ObjectWin0, ObjectWin1, ObjectWin2 and Projects ObjectWinLibX, where the UI is presented as the library. These variants show the phases of the modification of the code.
Functionality
  • The behavior of applications of the Release 12 is the same as the behavior of applicaons of the Release 11
  • The new test model of the Player Computer00 added. The only purpose of the creation of the model of the player Computer00 is to simulate the behavior of the application on the large amount of moves, which players make.

Release 13 (Folder PolyLine13)

The set of projects

The Release 13 contains the set of strongly dependent projects - the project of the main application and projects of three DLLs, where DLL has the special project to build it.

The set of projects contains:

  • The Project ModelsDLL (generates the DLL), where models of the computer strategies of the game Computer0, Computer1, Computer2 are placed.
  • The Project Computer3DLL (generates the DLL), where the model of the strategy Computer3 is placed.
  • The Project HumanInterfaceDLL (generates the DLL), where classes related to the HumanInterface are placed. This DLL is called using the library, which used while building the main application.
  • The Project Polyline13_ObjWindows - the project of the main executable application
  • The same functionality
Functionality

The behavior of applications of the Release 13 is the same as the behavior of applications of the Release 11, 12. The only difference is the DLL-related violations exceptions reacting.

Release 14 (Folder PolyLine14)

The set of projects

The Release 14 contains the set of projects, which demonstrates the different phases of the transformation of the application Polyline Release 13 to the application based on the Pzl-Technology (see http://wiki.visual-prolog.com/index.php?title=Category:VpPuZzle).

The set of projects contains:

  • The Project MonoApplicaion\Polyline14_ObjWindows_Mono.vipprj shows how classes are converted to pzl-Components.
  • The Project MonoApplicaionLib\Polyline14_ObjWindowsLib.vipprj is the same project as MonoApplicaion\Polyline14_ObjWindows_Mono.vipprj, where HumanInterface and player Computer3 represented as the libraries.
  • The directory pzlBasedApplication contains again the set of projects - the main application Polyline14_ObjWindowsPzl.vipprj and the set of projects, where each project corresponds to the pzlContainer, where basic components of the application are placed.
Functionality

The behavior of applications of the Release 14 is the same.
