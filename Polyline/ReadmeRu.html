Мотивация

Набор проектов является примером эволюции разработки приложения. Функциональной идеей приложения является простая игра. Точное имя игры нам неизвестно, поэтому она получила название Ломаная Линия ( Polyline).

Каждый этап разработки представляется отдельной реализацией (Релиз), что позволяет шаг за шагом наблюдать за развитием проекта. Каждая реализация расширяет функциональные возможности проекта и/или приводит к модификации кода. Внимательное изучение кода каждой реализации позволяет осваивать технику использования языка системы программирования Visual Prolog и изучать свойства и возможности этого языка и компонентов набора средств VipSpb SDK.

Описание игры

Изображается решетка для игры размером MxL точек.

Правила игры

Kаждый играющий по очереди добавляет координаты новой точки, являющейся продолжением ломаной линии в горизонтальном или вертикальном направлениях.

Получающаяся в результате траектория игры должна быть непрерывной, причем каждый последующий ход можно делать с ее любого конца.

Игрок, который "замыкает" траекторию (т.е. ходит в точку на существующей ломаной), выигрывает.

Прежде чем начать

Для построения изучения и изучения серии проектов небходимо установить стандартным образом систему программирования Visual Prolog.

Справочная информация, связанная с версией Visual Prolog помещена в файле VersionReference.html.

Комментарии к построению проектов

        Проекты релизов 1-10 являются консольными приложениями, на которых можно видеть применение концепции логического программирования без отвлечения на технику работы с GUI..
        Начиная с релиза 11 появляется приложение в технике GUI (PolylineXX_ObjWindows_XX.vipprj  Если не интересует динамика разработки проекта, Начиная с Релиз 11 представляет ряд проектов, которые имеют единую логику игры и управления, но различаются видом пользовательского интерфейса - Соnsole, ObjectConsole, Windows.
  1. Проекты Релизов 1-9 могут быть построены непосредственно без каких-либо дополнительных действий.
  2. Начиная с релиза 10, все проекты используеют переменную среды программировния IDE SpbSolutions, которая должна указывать директорию SpbVipTools.
  3. Начиная с Релиза 11 некоторые проекты демонстрируют технику построения библиотек (*.lib), поэтому необходимо иметь программу lib.exe от фирмы Microsoft, помещенную в директорию 3P_Tools\MSTools (детальная информация содержится в файлах Readme.html соответствующих релизов).
  4. Проекты, начиная с Релиз 13, демонстрируют и используют технику создания и применения DLL. Такие проекты могут строиться только с помощью Visual Prolog CE (см. детали в файле VersionReference.html).
  5. Проекты Релиза 14, помещенные в директорию PzlBasedApplicaions требуют выполнения процедуры регистрации компонентов после построения проектов директории Containers. Это должно быть сделано с использованием командного файла Programs\pzlBasedApplication\RegisterComponents.cmd.

Начальная идея взята из материлов форума по языку Пролог

Автором кода является Елена Ефимова.

Релиз 2 (Проект PolyLine2)

  • Если первый ход компьютерной модели, то он делается случайным образом
  • Модели человека и компьютера являются классами, генерирующими объекты
  • Предикаты, относящиеся к пользовательскому интерфейсу сгруппированы в классе HumanInterface

Релиз 3 (Проект PolyLine3)

  • Может быть более двух игроков
  • Тип каждого игрока (компьютер или человек) выбирается

Релиз 4 (Проект PolyLine4)

  • Компьютерная модель: если безопасный ход не найден, то он делается случайным образом (возможно крайне неудачным)
  • Структура класса computer существенно модифицирована

Релиз 5 (Проект PolyLine5)

  • Размер поля в коде установлен 10 x 10
  • Компьютерная модель: делается прогноз на K шагов вперед
  • Компьютерная модель: параметр глубины прогноза устанавливается

Релиз 6 (Проект PolyLine6)

  • Возможность включения в игру более двух игроков, каждый из который может иметь свою стратегию
  • Размер игрового поля задается
  • Простое добавление новых игроков
  • Режим многократных игр (рекомендуется только для компьютерных моделей)

    Построение и запуск

  • Открыть проект, построить и запустить на выполнение.
  • Язык приложения определяется помещением в командной строке идентификатора языка:
    • ru - русский
    • en - английский
  • При запуске из среды IDE параметр языка помещается в поле формы "Project Settings\Run Options\Run Arguments", получаемой из меню "Project\Settings..."

Релиз 12 (Семейство проектов)

Набор проектов

Набор проектов Релиз 12 отличается от того, что входил в состав Релиз 11:

  • В Релиз 12 проектов Console, ConsoleLib, ObjectConsoleLib нет (и не предполагается их поддержка в будущем).
  • Проект ObjectWin содержит три варианта ObjectWin0, ObjectWin1, ObjectWin2, демонстрирующие последовательность модификации кода.
  • Проект ObjectConsole содержит только окончательное решение и служит для проверки совместимости и работоспособности.

Функциональность

  • Функционально Релиз 12 ничем не отличается от Релиз 11.
  • Добавлен тестовый игрок - Computer00
    В целях проверки устойчивости проекта на играх большой размерности создан специальный игрок Computer00, который всегда делает один и тот же (специальный) ход. Модуль JuniourJudge, обрабатывающий ходы игроков, распознает этот ход, как специальный, не отображает его на игровом поле и позволяет делать следующий ход.

Релиз 13 (Семейство проектов)

Набор проектов

Релиз 13 состоит из набора строго взаимосвязанных проектов - проекта главного приложения и трех DLL, для каждой из которых предусмотрен самостоятельный проект:

  • Проект ObjectWin - проект главного исполняемого приложения.
  • Проект ModelsDLL (генерирует DLL), в который помещены модели компьютерных стратегий игры Computer0, Computer1, Computer2.
  • Проект Computer3DLL (генерирует DLL), в который помещена новая модель стратегии игры - Computer3.
  • Проект HumanInterfaceDLL (генерирует DLL), в который помещены классы, отноящиеся к HumanInterface. Эта DLL вызывается с помощью библиотеки, которая связывается с главным приложением в процессе построения приложения.

Функциональность

Функционально Релиз 13 ничем не отличается от Релиз 12.

Релиз 14 (Семейство проектов)

Набор проектов

Релиз 14 содержит набор проектов, которые демонстрируют различные фазы преобразования приложения Polyline Релиз 13 в приложение, выполненное на базе Pzl-Технологии (описание технологии приведено здесь).

Набор проектов содержит:

  • Проект MonoApplicaion\Polyline.prj6 показывающий как классы преобразуются в pzl-Компоненты
  • Проект MonoApplicaionLib\Polyline.prj6 является повторением проекта MonoApplicaion\Polyline.prj6 с тем отличием, что модуль HumanInterface и модуль Computer3 представлены библиотеками.
  • Раздел pzlBasedApplication содержит в свою очередь набор проектов - главное приложение Polyline.prj6 и набор проектов, где каждый проект представляет pzl-Контейнер. Базовые компоненты приложения помещены в соответствующие контейнеры.

Функциональность

Поведение приложения Релиза 14 ничем не отличается от релиза 13.
