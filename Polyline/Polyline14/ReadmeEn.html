Tutorial Project. Release 14

The Goal

The demonstration of the implementation of the communication of the main application with parts of the application, which are placed to the special kind of DLL (Dynamically Linked Library) - PzlContainers.

PzlContainers contains PzlComponents. Both PzlContainers and PzlComponents fit the agreements of the Pzl Technology (Vp Puzzle Technology).
The detailed description of the VpPuZzle technology can be seen at http://wiki.visual-prolog.com/index.php?title=Category:VpPuZzle.

The Code

The Release 14 contains the set of projects, which demonstrates the different phases of the transformation of the application Polyline Release 13 to the Pzl-based application.

The set of projects contains:

  1. The Project MonoApplicaion\Polyline.vipprj shows how classes are converted to pzl-Components. Here the wrapping approach been used.
  2. The Project MonoApplicaionLib\Polyline.vipprj is the same project as the MonoApplicaion\Polyline.vipprj, but HumanInterface and player Computer3 represented as the libraries.
  3. The directory pzlBasedApplication contains again the set of projects - the main application Polyline.vipprj and the set of projects, corresponding to containers (see version dependant details in the file VersionReference.html)
    • Containers\Competitors\Competitors.vipprj
    • Containers\PolylineBlackHorsePlayers\PolylineBlackHorsePlayers.vipprj
    • Containers\PolylineGame\PolylineGame.vipprj
    • Containers\PolylineGameStatistics\PolylineGameStatistics.vipprj
    • Containers\PolylineHumanInterface\PolylineHumanInterface.vipprj
    • Containers\PolylineJuniourJudge\PolylineJuniourJudge.vipprj
    • Containers\PolylinePublicPlayers\PolylinePublicPlayers.vipprj
    • Containers\PolylineSeniourJudge\PolylineSeniourJudge.vipprj

Functionality

The behavior of applications of the Release 14 is the same as the behavior of applications of the Release 13

Run notes

  • The execuatable applications correspond to projects:
    • Programs\MonoApplicaion\Polyline14_ObjWindows_Mono.vipprj or Programs\MonoApplicaionLib\Polyline.vipprj, where all components are in one application
    • Programs\pzlBasedApplicationPolyline14_ObjWindowsPzlvipprj, which communicates with components placed to pzl-Containers and registered in the PzlRegistry.pzr file. The file PzlRegistry.pzr must be in the same directory as the executable file
  • The language of the application may be defined in the command line by the language identifier:
    • en - English
    • ru - Russian
  • When running the program from the IDE, the parameter of the language must be placed to the field Project Settings\Run Options\Run Arguments, available from menu Project\Settings...
